Fundamentação Teórica: O Grupo de Permutações e a Ordem de SJT

A geração de permutações é um problema central na matemática computacional, fundamentado na teoria de grupos, onde o conjunto de todas as permutações de n elementos forma o Grupo Simétrico Sn. Tradicionalmente, algoritmos lexicográficos exigem rearranjos estruturais complexos, mas a classe de algoritmos de Mudanças Mínimas (Plain Changes) busca uma trajetória hamiltoniana no grafo de Cayley, onde cada vértice (permutação) é alcançado através de uma única transposição adjacente.


O Algoritmo T apresenta-se como uma implementação otimizada e melhorada do Algoritmo P (Steinhaus-Johnson-Trotter). Enquanto o Algoritmo P baseia-se na busca exaustiva do maior elemento móvel e na atualização global de vetores de direção em cada passo, o Algoritmo T refina esse processo ao focar na transição.

Essa evolução permite que o Algoritmo T opere em duas frentes distintas:

  • Modo Online: Onde as transições são calculadas Just-In-Time, mantendo um estado de memória reduzido (O(n)).
  • Modo Offline: Onde a inteligência do Algoritmo P é utilizada para pré-calcular uma Tabela de Transações, permitindo que a geração subsequente ocorra em tempo constante (O(1)) por permutação, eliminando completamente a latência de decisão algorítmica.

How Algorithm T Works

Algorithm T is split into two simple functions. The offline phase builds a swap plan once. The online phase reuses that plan to generate permutations quickly.

Pseudocode - T Online

# Inicialização do Estado de Controle
c ← [0, ..., 0] # Contadores de "movimento" para cada nível
d ← [-1, ..., -1] # Bússola para mudanças adjacentes
i ← 0

enquanto (i < n):
se (c[i] < i):
# 1. Localização e Troca (Onde o custo Online ocorre)
alvo ← encontrar_indice(i + 1) # Busca O(n)
vizinho ← alvo + d[alvo]
trocar(a[alvo], a[vizinho])

# 2. Atualização do Contador e Reset do Ponteiro
c[i] ← c[i] + 1
i ← 0 # Reinicia para garantir a menor troca disponível
senão:
# 3. Inversão de Direção e Subida de Nível
d[i] ← -d[i]
c[i] ← 0
i ← i + 1

Pseudocode - T Offline

# FASE 1: Geração da Tabela de Transições (Setup)
CONSTRUIR_TABELA(n):
plano ← [ ] # Lista de tuplas (i, j)
Executar lógica do T_Online sobre vetor [1..n]
Para cada troca identificada entre índices i e j:
    Adicionar (i, j) ao plano
retornar plano
# FASE 2: Execução de Alta Performance (Uso Real)
T_OFFLINE(vetor):
plano ← CONSTRUIR_TABELA(tamanho do vetor)
exibir vetor # Permutação inicial

para cada (i, j) em plano:
trocar(vetor[i], vetor[j]) # Custo O(1)
exibir vetor

Online Version

Esta versão recebe o plano pré-computado e aplica cada troca em ordem. Ela não recalcula a lógica da próxima jogada; apenas segue o plano.

Offline Version

Esta versão prepara o plano de troca. Ela encontra quais posições adjacentes devem ser trocadas para passar de uma permutação para a próxima. O resultado é um roteiro compacto.

Step by Step

1. Start with the identity permutation.
2. Build the swap plan offline.
3. Reuse that plan online to generate every new permutation.
4. Measure offline and online costs separately.
5. Compare T with P.
